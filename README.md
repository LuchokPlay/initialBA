# Анализ бизнес-процессов из Excel

Программа для анализа бизнес-данных из Excel файлов с автоматическим определением структуры и преобразованием в JSON формат.

## Быстрый запуск без установки Python

Чтобы воспользоваться программой, можно просто скачать файл `BusinessProcessAnalyzer2.exe` из корня репозитория и запустить его на Windows.

## 🚀 Быстрый старт

```bash
# Установка зависимостей
cd программа
pip install -r requirements.txt

# Запуск приложения
python main.py
```

## 📋 Возможности

### ✨ Парсер Excel файлов
- **Автоматическое определение структуры** - программа сама находит нужные столбцы по названиям
- **Гибкая обработка** - работает с файлами, где столбцы называются по-разному
- **Оптимизированный JSON** - действия хранятся в отдельном списке с использованием ID
- **Проверка данных** - валидация формата Case_ID и дат

### 🎨 Пользовательский интерфейс
- **Современный дизайн** на PyQt5
- **Предварительный анализ** файла перед парсингом
- **Фоновый парсинг** с индикатором прогресса
- **Детальная информация** о результатах
- **Обработка ошибок** с понятными сообщениями
- **Автоматическое сохранение JSON** - результат сохраняется в файл после парсинга
- **Ручное сохранение** - возможность сохранить JSON в любое место
- **Модуль статистики** - комплексный анализ временных и качественных метрик процессов

## 💾 Сохранение результатов

После успешного парсинга программа автоматически сохраняет JSON файл рядом с исходным Excel файлом. Имя файла формируется по шаблону: `{имя_excel_файла}_parsed.json`.

### Дополнительные возможности:
- **Кнопка "Сохранить JSON"** - позволяет сохранить результат в любое место на диске
- **Полный JSON** - сохраняются все записи, метаданные и статистика анализа
- **UTF-8 кодировка** - файл сохраняется в UTF-8 для корректного отображения кириллицы

## 📊 Формат данных

### Входной Excel файл
Ожидается таблица с минимум тремя столбцами:
- **ID кейса** (примеры названий: "Case ID", "ID кейса", "Кейс")
- **Действия** (примеры названий: "Статус", "Действие", "Операция")
- **Дата/время** (примеры названий: "Дата начала", "Время", "Дата и время")

Пример структуры:
| Case ID   | Статус     | Дата начала     |
|-----------|------------|-----------------|
| Case_123  | Создан     | 02.05.2021  6:44:00 |
| Case_123  | Обработан  | 02.05.2021  7:15:00 |
| Case_124  | Создан     | 03.05.2021  9:30:00 |

### Выходной JSON формат
```json
{
  "actions": ["Создан", "Обработан", "Завершен"],
  "records": [
    {
      "case_id": 123,
      "case_raw": "Case_123",
      "action_id": 0,
      "datetime": "2021-05-02T06:44:00"
    }
  ],
  "metadata": {
    "total_records": 150,
    "unique_actions": 3,
    "columns_detected": {
      "case_id_column": "Case ID",
      "action_column": "Статус",
      "datetime_column": "Дата начала"
    }
  }
}
```

## 🏗️ Архитектура

```
├── main.py                 # Главный файл запуска
├── excel_parser.py         # Модуль парсинга Excel
├── sequence_analyzer.py    # Анализ последовательностей действий
├── statistics_analyzer.py  # Статистический анализ процессов
├── ui_main.py             # Модуль пользовательского интерфейса
├── requirements.txt        # Зависимости
└── README.md              # Документация
```

### excel_parser.py
Основные функции:
- `parse_excel_to_json()` - парсинг файла в JSON с анализом последовательностей
- `get_excel_info()` - анализ структуры файла
- `detect_column_indices()` - автоматическое определение столбцов

### sequence_analyzer.py
Интеллектуальный анализ процессов:
- `SequenceAnalyzer` - анализатор последовательностей действий
- `find_start_actions()` - определение стартовых действий
- `find_end_actions()` - определение финальных действий
- `_find_dead_ends()` - обнаружение тупиковых действий
- `find_most_likely_sequence()` - построение логической последовательности

### statistics_analyzer.py
Комплексный статистический анализ:
- `StatisticsAnalyzer` - анализатор статистики процессов
- `get_process_statistics()` - метрики процессов (длительность, количество действий)
- `get_action_statistics()` - метрики действий (частота, время выполнения)
- `get_time_distribution()` - распределение процессов по времени
- `get_process_flow_analysis()` - анализ потока и переходов
- `get_quality_metrics()` - метрики качества и эффективности

### ui_main.py
Компоненты интерфейса:
- Выбор файла через диалог
- Отображение информации о файле
- Фоновый парсинг с прогрессом
- Форматированное отображение результатов

## 📊 Модуль статистики

Программа включает мощный модуль статистического анализа, который рассчитывает комплексные метрики на основе JSON данных процессов.

### Возможности статистики

#### ⏱️ Временные метрики
- **Длительность процессов**: среднее, минимальное, максимальное время
- **Время выполнения действий**: статистика по каждому действию
- **Распределение по интервалам**: динамический анализ продолжительности процессов
- **Стандартные отклонения**: вариабельность выполнения

#### 🔄 Анализ потока
- **Частота действий**: наиболее и наименее используемые действия
- **Переходы между действиями**: наиболее вероятные последовательности
- **Топ переходов**: самые частые комбинации действий

#### 🚨 Поиск сбоев
- **Незавершенные процессы**: процессы, не завершенные последним действием из списка actions
- **Анализ причин**: сравнение последнего действия процесса с ожидаемым конечным
- **Временные метки**: даты начала и последнего действия
- **Детальная информация**: ID процесса, последнее действие, ожидаемое действие, количество действий

### Использование программы

#### Парсинг Excel файлов
1. **Выберите Excel файл** с помощью кнопки "Выбрать Excel файл"
2. **Нажмите "Парсить данные"** для обработки файла
3. **Файл автоматически сохранится** в формате JSON
4. **Перейдите к статистике** с помощью кнопки "Статистика>>>" (справа в панели)

#### Работа со статистикой
1. **Просмотрите данные на вкладках**:
   - **Общая статистика**: основные метрики процессов и распределение по времени
   - **Статистика действий**: детальная таблица с метриками по каждому действию
   - **Переходы**: анализ наиболее частых переходов между действиями
   - **Поиск сбоев**: анализ незавершенных процессов
2. **Нажмите "<<<Парсинг"** для возврата к результатам парсинга

#### Загрузка готовых JSON файлов
- **Нажмите "Выбрать готовый JSON"** для загрузки уже обработанного файла
- **Посмотрите информацию о файле** (структура, первые 1000 записей)
- **Перейдите к статистике** с помощью кнопки "Статистика>>>"

### Интерфейс статистики

Статистика отображается в удобном табличном виде с четырьмя основными разделами. В режиме статистики полностью скрыты все элементы управления парсингом и информационные панели - доступно только содержимое статистики и кнопка возврата к парсингу в левом углу.

- **Общая статистика**: сводная информация о процессах и их распределении по времени выполнения
- **Статистика действий**: таблица со столбцами Действие, Выполнений, Процессов, Средняя длительность, Мин/Макс время
- **Переходы**: списки наиболее частых действий и переходов между ними
- **Поиск сбоев**: анализ незавершенных процессов с указанием даты начала, последнего действия, ожидаемого действия и количества выполненных действий

pip install PyQt5 pandas openpyxl

## 📖 Использование

1. **Запустите приложение**: `python main.py`
2. **Выберите файл**: Нажмите "Выбрать Excel файл" и укажите путь к файлу
3. **Проверьте структуру**: Программа автоматически определит столбцы и покажет информацию
4. **Запустите парсинг**: Нажмите "Парсить данные"
5. **Просмотрите результаты**: В нижней части окна отобразятся обработанные данные

## ⚠️ Обработка ошибок

Программа обрабатывает следующие типы ошибок:
- **Неверная структура файла** - отсутствие необходимых столбцов
- **Некорректный формат Case_ID** - значение не содержит числовой части
- **Ошибки даты/времени** - значение не соответствует поддерживаемым форматам даты/времени
- **Проблемы чтения файла** - поврежденный файл или недостаточные права
